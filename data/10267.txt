Microsoft announces Windows DNS Server Heap Overflow Vulnerability, users dissatisfied with patch details
11th December was Microsoft’s December 2018 Patch Tuesday, which means users had to update their computers to be protected from the latest threats to Windows and Microsoft products. | PatchVulnerability
Microsoft has fixed 39 vulnerabilities, with 10 of them being labeled as Critical. | PatchVulnerability
Keeping up with its December 2018 Patch Tuesday, Microsoft announced on its blog that a vulnerability exists in Windows Domain Name System (DNS). | DiscoverVulnerability
There was not much information provided to the customers about how and when this vulnerability was discovered. | DiscoverVulnerability
The following details were released by Microsoft: | O
The Exploit | O
Microsoft Windows is prone to a heap-based buffer-overflow vulnerability. | DiscoverVulnerability
A remote code execution vulnerability exists in Windows Domain Name System (DNS) servers when they fail to properly handle requests. | DiscoverVulnerability
An attacker who successfully exploits this issue may execute arbitrary code within the context of the affected application. | DiscoverVulnerability
Microsoft states that failed exploit attempts will result in a denial-of-service condition. | O 
Windows servers that are configured as DNS servers are at risk from this vulnerability. | O
