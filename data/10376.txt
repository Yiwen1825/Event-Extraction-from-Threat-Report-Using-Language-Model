Microsoft fixes 62 security flaws on Patch Tuesday and re-releases Windows 10 version 1809 and Windows Server 2019
Yesterday, on Microsoft’s Patch Tuesday the company released its monthly security patches that fixed 62 security flaws. | PatchVulnerability
These fixes also included a fix for a zero-day vulnerability that was under active exploitation before these patches were made available. | PatchVulnerability
Microsoft also announced the re-release of its Windows 10 version 1809 and Windows Server 2019. | O
Microsoft credited Kaspersky Lab researchers for discovering this zero-day, which is also known as CVE-2018-8589 and impacts the Windows Win32k component. | DiscoverVulnerability
A Kaspersky spokesperson told ZDNet, “they discovered the zero-day being exploited by multiple cyber-espionage groups (APTs).” | DiscoverVulnerability
The zero-day had been used to elevate privileges on 32-bit Windows 7 versions.  | DiscoverVulnerability
This is the second Windows elevation of privilege zero-day patched by Microsoft discovered by Kaspersky researchers. | DiscoverVulnerability
Last month, Microsoft patched CVE-2018-8453, another zero-day that had been used by a state-backed cyber-espionage group known as FruityArmor. | PatchVulnerability
However, in this month’s Patch Tuesday, Microsoft has not patched a zero-day that is affecting the Windows Data Sharing Service (dssvc.dll). | PatchVulnerability, DiscoverVulnerability
This zero-day was disclosed on Twitter at the end of October. | DiscoverVulnerability
According to ZDNet, “Microsoft has published this month a security advisory to instruct users on how to properly configure BitLocker when used together with solid-state drives (SSDs).” | O
As reported by Microsoft, the Windows 10 October 2018 update caused user’s data loss post updating. | O
Due to this, the company decided to pause the update. | O
However, yesterday, Microsoft announced that it is re-releasing Windows 10 version 1809. | O
John Cable, the director of Program Management for Windows Servicing and Delivery at Microsoft said, “the data-destroying bug that triggered that unprecedented decision, as well as other quality issues that emerged during the unscheduled hiatus, have been thoroughly investigated and resolved.” | O
Microsoft also announced the re-release of Windows Server 2019, which was affected by the same issue. | DiscoverVulnerability
According to ZDNet, “The first step in the re-release is to restore the installation files to its Windows 10 Download page so that “seekers” (the Microsoft term for advanced users who go out of their way to install a new Windows version) can use the ISO files to upgrade PCs running older Windows 10 versions.” | O
Michael Fortin, Windows Corporate Vice President, in a blog post, offered some context behind the recent issues and announced changes to the way the company approaches communications and also the transparency around their process. | O 
Per Fortin, “We obsess over these metrics as we strive to improve product quality, comparing current quality levels across a variety of metrics to historical trends and digging into any anomaly.”| O
To know more about this in detail, visit Microsoft’s official blog post. | O
